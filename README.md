# MLB Daily Prediction System

Predicts three betting markets for every game on today's MLB slate:

| Market | Model | Output |
|---|---|---|
| Moneyline | XGBClassifier | Winner + confidence % (`predict_proba`) |
| Total runs | 3x XGBRegressor | Away runs, home runs, combined total |
| NRFI / YRFI | XGBClassifier | Call + confidence % (`predict_proba`) |

## Setup

```bash
pip install -r requirements.txt
python mlb_predictor.py
python mlb_predictor.py --date 2026-07-12
```

If `xgboost` is not installed the script silently falls back to
scikit-learn Random Forests.

## Daily prediction pipeline (`mlb_predictor.py`)

1. **Schedule** — `statsapi.schedule()` gives game_pk, teams, probable
   starters; `statsapi.lookup_player()` resolves MLBAM pitcher IDs;
   the live boxscore gives batting orders when lineups are posted.
2. **One bulk Statcast download** — a single `pybaseball.statcast()`
   call covering the last 90 days. Every feature below slices this
   in-memory frame (microseconds per pitcher/team), replacing the old
   `statcast_pitcher()` loop that took 30–60 s per pitcher cold.
   **No FanGraphs requests are made anywhere** (they were
   Cloudflare-blocked on some networks).
3. **Moneyline features** — starter last-30-day K/9, BB/9, GB%, and
   **calculated FIP** = `(13*HR + 3*(BB+HBP) - 2*K)/IP + 3.15`;
   team offense = rolling 30-day Statcast wOBA **split vs the opposing
   starter's handedness** (`p_throws`), indexed to the same-window,
   same-hand league wOBA (wRC+ proxy) plus the matching split OBP.
   Falls back to the all-hand window (< 120 platoon PA), then to MLB
   Stats API season numbers.
4. **Totals features** — starters' expected runs allowed (FIP proxy),
   last-14-day bullpen **RA/9** from the bulk frame (relievers = every
   pitcher except each game's starter), and park factors (embedded
   constants).
5. **NRFI features** — 1st-inning-only slices: RA/9 from score deltas
   per pitch, 1st-inning walk rate, plus season OBP/ISO for batters 1–3
   of each posted lineup. RA/9 (not ERA) is deliberate: an unearned run
   cashes a YRFI bet exactly like an earned one.

Every fetch has a league-average fallback so an unannounced starter or
an API timeout never kills the slate.

## Building the training data (no lookahead bias)

`mlb_history_2024_2025.csv` is generated by a three-step pipeline. Every
feature for a game on date **D** is computed from data in a window that
ends on **D-1** — never from end-of-season totals, which would leak the
future into training and make backtests look brilliant while the model
bleeds on live slates.

### Step 1 — schedule backbone: `scrape_bref_schedules.py`

```bash
python scrape_bref_schedules.py            
python scrape_bref_schedules.py --teams ARI --years 2024  
```

Scrapes each team's Baseball-Reference *schedule-scores* page (e.g.
`/teams/ARI/2024-schedule-scores.shtml`) with **pydoll** driving real
Chrome — this passes B-Ref's Cloudflare check and pages are fetched
3.5 s apart to respect the 20 req/min limit. Only these 60 pages are
scraped (not thousands of player pages); they provide the targets:
date, home/away, `home_win`, `home_runs`, `away_runs`, `total_runs`.
Output: `data/bref/{TEAM}_{YEAR}.csv` + `data/schedule_backbone.csv`
(one row per game; reruns skip already-scraped pages).

### Step 2 — bulk Statcast: `download_statcast.py`

```bash
python download_statcast.py                
```

One league-wide pitch-level download per season (~712k pitches each),
trimmed to the ~20 needed columns → `data/statcast_{year}.parquet`
(~2.4 MB each). All features derive from these files.

### Step 3 — chronological features: `build_history.py`

```bash
python build_history.py
```

Matches every backbone game to its Statcast `game_pk` (doubleheaders
disambiguated by final score), derives starters from the first pitch of
each 1st-inning half, then computes per game, **as of the day before**:

| Feature group | Window | Source |
|---|---|---|
| Starter K/9, BB/9, GB%, FIP | last 30 days | Statcast events |
| Team wRC+ proxy & OBP vs starter hand | last 30 days | Statcast wOBA (`p_throws` split) |
| Bullpen RA/9 (`*_bullpen_era14`) | last 14 days | Statcast score deltas, non-starters |
| Starter 1st-inning RA/9, BB rate | last 90 days | Statcast inning-1 slices |
| Top-3 lineup OBP / ISO | season to date | Statcast events (resets each season) |
| `first_inning_run` (YRFI target) | that game | Statcast inning-1 score deltas |
| Win/runs targets | that game | B-Ref backbone |

Rolling windows use per-key cumulative sums (`RollingLookup`), so the
full 4,859-game build takes about a minute. Early-season games fall back
to league-average values where a window is empty. If the CSV is missing,
`mlb_predictor.py` trains on synthetic data and prints a loud warning —
useful for testing the pipeline, useless for betting.

### CSV schema

**Features**
`home_sp_k9, home_sp_bb9, home_sp_xfip, home_sp_gb_pct,
away_sp_k9, away_sp_bb9, away_sp_xfip, away_sp_gb_pct,
home_team_wrc_plus, home_team_obp, away_team_wrc_plus, away_team_obp,
home_sp_xra9, away_sp_xra9, combined_sp_xra9,
home_bullpen_era14, away_bullpen_era14, park_factor,
home_sp_fi_ra9, home_sp_fi_bb_rate, away_sp_fi_ra9, away_sp_fi_bb_rate,
home_top3_obp, home_top3_iso, away_top3_obp, away_top3_iso`

(The `*_xfip` / `*_era14` names are kept for schema compatibility; they
now hold calculated FIP and reliever RA/9 respectively — computed the
same way in training and live prediction, both from Statcast, via the
shared constants in `mlb_common.py`.)

**Targets**
`home_win` (0/1), `home_runs`, `away_runs`, `total_runs`,
`first_inning_run` (0/1 — 1 means YRFI)

Plus identifier columns (`date, season, game_pk, home_team, away_team,
home_sp_id, away_sp_id`) that the trainer ignores.

## Honest holdout numbers (2024–25, 4,859 games)

- Moneyline accuracy **0.550** (always-home baseline: 0.532)
- Total-runs MAE **~3.5 runs**; per-team MAE ~2.5
- NRFI/YRFI accuracy **~0.49** (YRFI base rate 0.484 — first-inning
  scoring is close to a coin flip; use the probability, not the label)

These are what leak-free numbers look like. Anything dramatically
better on this feature set usually means lookahead bias.

## Known limitations

- The wRC+ proxy is wOBA indexed to league average — it is not
  park-adjusted the way real wRC+ is.
- Bullpen and 1st-inning "ERA" are really RA/9 (unearned runs
  included). For NRFI that is a feature, not a bug.
- Savant team codes differ from B-Ref (`AZ/ARI`, `CWS/CHW`, and the
  Athletics are retroactively `ATH` even for Oakland 2024) — handled in
  the mapping tables.
- Park factors are embedded constants (no stable pybaseball endpoint);
  Sacramento's Sutter Health Park is estimated at 1.05.
- This is a modeling exercise, not betting advice.
